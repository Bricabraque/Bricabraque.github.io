Bricabrac : how to git

jeu. 23 février 2017
fr | en

Depuis peu de temps, je me suis inscrit sur Github afin de contribuer à la traduction de divers projets de logiciels libres. Bien que je connaissais déjà le principe, l'utilisation propre du logiciel de gestion de versions git m'a toujours semblé complexe et tiré par les cheveux. Mais ces derniers jours, je m'y suis mis sérieusement et j'apprends petit à petit à me familiariser avec les diverses commandes.

Je présenterai ici un pense-bête du workflow que j'utilise quand je contribue à des projets dont je ne suis pas l'initiateur; quasiment tous, donc.

Fork et clonage du dépôt distant

Après avoir forké le dépôt original du projet concerné sur la plateforme Github, il faut cloner le dépôt du fork en local.

cd chemin/de/votre/dossier/de/travail
git clone <adresse-du-fork>
cd <dossier-du-fork>

On ajoute un dépôt distant qu'on nommera upstream avec l'adresse du dépôt upstream.

git remote add upstream <adresse-du-dépôt-upstream>

Si on vérifie la configuration des dépôt distants avec la commande git remote -v, on devrait avoir ça :

origin  https://fork.git (fetch)
origin  https://fork.git (push)
upstream    https://projet-original.git (fetch)
upstream    https://projet-original.git (push)

Faire des modifications

Voici les étapes que suivront les modifications faites.

On met à jour depuis le dépôt upstream et on crée une nouvelle branche pour le futur commit :

git fetch upstream
git branch <nouvelle-branche>

On ajoute les fichiers à pousser dans la liste d'attente :

git add <fichier-modifié>

On commit :

git commit -m "Message du commit"

On pousse le commit sur le dépôt du fork, qui nous appartient :

git push origin <nouvelle-branche>

On termine en retournant sur Github pour faire une pull request depuis le fork vers le dépôt upstream.

Synchronisation du fork

Imaginons qu'à ce stade, la pull request a été fusionnée dans le dépôt upstream par son propriétaire. On va maintenant synchroniser le fork avec le dépôt upstream pour être jour dans les commits.

On récupère les mise à jour depuis upstream :

git fetch upstream

On retourne dans la branche master :

git checkout master

On fusionne la branche upstream/master avec la branche master locale :

git merge upstream/master

On pousse tout ça sur le fork distant :

git push origin master
