Bricabrac : how to git

jeu. 23 février 2017
en | fr

For a few days, I have signed up for a Github in order to contribute to the translation of diverse free-software projects. Even though I have already known its main idea, the proper using of the version control system git has always seemed to me as a complex and far-fetched process. However, I have seriously dug into it these least few days and have been gradually getting used to the various commands.

I will draw here a brief reminder of the workflow that I have learnt and that I currently use when I contribute to a project which I am not the owner; that is, nearly all of them.

Forking et cloning the remote repository

After forking the original project repository on Github, we need to clone the fork repo locally.

cd path/of/your/working/dir
git clone <fork-url>
cd <fork-dir>

Then we add a remote that will be named upstream with the upstream repo's url.

git remote add upstream <upstream-repo-url>

If we check the remots with git remote -v, the terminal should return these lines :

origin  https://fork.git (fetch)
origin  https://fork.git (push)
upstream    https://upstream-repo.git (fetch)
upstream    https://upstream-repo.git (push)

Making changes

Here are the steps which follow the changing.

We update locally from the upstream repo and we create a new branch for the commit-to-be :

git fetch upstream
git branch <new-branch>

We add the modified files to push to the commit queue :

git add <modified-file>

Commit :

git commit -m "Commit message"

We push the commit to the fork repo we own :

git push origin <new-branch>

We complete the operation by going back on Github to issue a pull request from the fork to the upstream repository.

Syncing a fork

At that stage, let us say the pull has been merged into the upstream repository by its owner. We now sync the fork with the upstream repo in order to be up-to-date.

We fetch the updates from upstream :

git fetch upstream

We go back to the master branch :

git checkout master

We merge the upstream/master branch with the local master branch :

git merge upstream/master

And we push all the stuff to the remote fork :

git push origin master
